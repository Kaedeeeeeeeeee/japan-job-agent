# Japan Job Agent v0.2 Foundation：v0.1 差异规格

状态：Accepted  
日期：2026-07-13  
基线：`japan-job-agent-spec-v0.1/`（只读）

## 1. 目的

v0.2 修复 v0.1 中可能造成错误关岗、事实不可追溯、解析重放污染原文、多租户公司关系失真和重复副作用的设计缺口。本规格是 v0.1 的增量修订；冲突处以 v0.2 和 ADR-013～018 为准。

## 2. 十项修订

1. **完整快照契约**：Connector 只返回一页事实。只有 Orchestrator 完成全部分页、租户身份和 provider total 校验后才可生成 `authoritative`。分页中断、解析错误、总数不符或熔断均生成 `partial`。
2. **Raw 与 Extraction 分离**：`source_job_versions` 只保存不可变原文字节的存储元数据、`raw_hash`、版本化规范化后的 `content_hash` 和 HTTP 元数据。任何 parser 信息和提取结果只能写入 `source_job_extractions`。
3. **多值与补充字段**：雇佣形式、在留资格、地点、语言、技能和薪资使用专项多值表。签证转签、搬迁支持/要求、调动要求、试用期薪资及 `is_calculated` 均可表达。
4. **Tenant/Company 解耦**：`source_instances` 不包含 Company 外键。公司只通过有生效区间和证据的 `company_source_relationships`、岗位归属关系关联。
5. **误关闭熔断**：上次 active 数量至少 5 且本次为零、缺失比例超过 50%、或单次缺失超过 25 时，快照降级为 `partial`，Source 变为 degraded，创建人工审核任务，缺失计数不变。
6. **缺失时间间隔**：只有成功且 finalized 的 `authoritative` 集合快照可增加缺失计数；两次计数间隔不得小于 `source_policies.minimum_missing_interval`。`single_record` 不能通过集合缺失关闭其他岗位。
7. **主来源单一事实源**：删除 Canonical Job 上的 primary 快捷外键。当前 primary 仅由 `canonical_job_sources` 的部分唯一索引表达；Canonical Version 的 materialization input 同样最多一个 primary。
8. **Temporal/Outbox 幂等**：同步以 `(source_instance_id,idempotency_key)` 去重，Activity 以 Temporal execution identity 和 activity key 去重。Outbox 以 `dedup_key` 去重，发布者使用租约和 `FOR UPDATE SKIP LOCKED`，消费者以 `(consumer_name,event_id)` 去重。
9. **质量指标拆分**：known rate 单独衡量完整度；evidence integrity 只计算非 unknown 事实中含证据的比例，禁止把 unknown 当作证据失败。
10. **SmartRecruiters 契约修正**：公开 Career Site API 使用 `api.smartrecruiters.com/v1/companies/{companyIdentifier}/postings`；受保护 Customer API 必须显式配置凭证且不得伪装为公开 ATS feed。第一阶段不实现 SmartRecruiters Connector，只保留经策略审核后的扩展点。

## 3. Snapshot 状态机

- `authoritative`：全部页完成；无解析错误；所有岗位属于同一 Source Instance；provider total（若提供）与去重后数量一致；未触发关闭熔断；Source Policy 允许完整快照。
- `partial`：上述任一条件失败，或收到 403、429、5xx、超时、Schema 变化、异常空响应。
- `single_record`：schema.org/手动单岗位输入。只能依据明确关闭文本、HTTP 410 或人工确认关闭该记录。

页面响应永远没有权限声明 authoritative。`fetchCollectionPage()` 返回 jobs、下一游标、`isLastPage`、可选 provider total 和响应元数据；`fetchRecord()` 接受不依赖 external ATS ID 的 `SourceJobIdentity`。

## 4. Hash 契约

- `raw_hash = SHA-256(raw bytes)`，任何字节差异均产生新值。
- `content_hash = SHA-256(canonicalization_version + NUL + canonicalized content)`。
- canonicalization 由平台实现并显式版本化，Connector 和 Parser 不得自行决定 hash。
- 相同 `(source_job_record_id, raw_hash)` 不新增 Raw Version；相同 `(raw version, parser key, parser version, schema version)` 不新增 Extraction。

## 5. 高风险字段与证据

雇佣形式、签证、语言、地点和薪资必须显式为 `known`、`unknown` 或 `conflicting`，不得用 SQL `NULL` 代替 unknown。每个非 unknown 事实必须引用当前 Extraction 的 Evidence，包含原文、Source URL 和定位信息。Parser 重放可新增 Extraction 和必要的 Canonical Version，但不得更新 Raw Version。
