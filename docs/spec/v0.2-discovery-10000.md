# v0.2 job-level Discovery expansion

## Acceptance boundary

The expansion is complete only when the database reports at least 10,000 valid job candidates, at least 2,000 active Canonical Jobs with a verified official source, zero formal strong-key duplicates, and a maximum source-family share of 40%.

A valid candidate has an independent detail URL, company, title, and deterministic Japan location. Official collections require an observation tied to a succeeded, complete authoritative Import Run no older than 72 hours. The database guard rejects authority unless all pages completed, tenant identity is consistent, provider totals agree, and parse errors are empty. Search and aggregator leads require two observations no older than 30 days. Discovery rows never participate in recommendations, formal missing-count reconciliation, or closure.

## Implemented sources

- SmartRecruiters, Lever, and Ashby public APIs use complete collection snapshots.
- Talentio uses its public sitemap as the authoritative tenant index and fetches exact job detail responses.
- YOLO JAPAN is a foreign-worker-oriented lead source. Category and prefecture collections provide two independent observations of each JSON-LD `JobPosting`.
- engage search sitemap entries are non-authoritative leads; sitemap presence and a successful JSON-LD detail fetch are recorded as two observations.
- Existing verified formal jobs are backfilled as `promoted` candidates without weakening their original Raw/Extraction/Canonical audit history.

The live collectors are resumable through stable observation keys, enforce HTTPS host allowlists, use retry with jitter, cap response sizes, and limit each host to one request per second. Aggregator candidates cannot be promoted directly.

## Date behavior

`published`, `sourceUpdated`, and `validThrough` use explicit `known`, `unknown`, or `conflicting` states. Values retain date or datetime precision and Evidence IDs in Extraction and Canonical tables. UI/API display the source publication date when known; otherwise they label `firstSeenAt` as first discovery and never present it as publication time.

## Promotion

Talentio promotion first verifies that the corporate HTTPS site or one of its bounded recruitment pages links to the exact Talentio tenant. Only then may Worker create a verified Company–Domain–Source relationship. A complete formal sync persists Raw objects locally, replays the current versioned deterministic Parser, creates field evidence, reconciles lifecycle, and materializes Canonical Jobs. Failure leaves the candidate isolated with a machine-readable `resolution_pending` reason.

The default deployment is PostgreSQL 16 + pgvector, Temporal, API, Worker, and Web on one 16 GB Linux host. Raw objects and backups use mounted local disk. Web/API bind only to a configured Tailscale address; GitHub Actions joins the tailnet with an ephemeral tagged identity for nightly readiness checks.
